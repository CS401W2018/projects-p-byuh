Engagement Questions March-28

  1. What is AJAX?

    AJAX (Asynchronous Javascript And XHTML) is a method to have your webpage send and recieve data in the background, allowing data and elements of your page to be updated without needing to reload the webpage entirely.

  2. Why do you put your JS Code in a separate file?

    It allows for the javascript to be easily used and shared by multiple pages on your website, preventing redundant code on your webpage. It also allows for more streamlined collaboration on websites, as you can have developers working on different files for the same webpage.

Engagement Questions March-24

  1. What is JavaScript? How did it get its name?

    JavaScript is a programming language used to add dynamic and interactive elements to a webpage. The developer's of JavaScript, Sun Microsystems, wanted their users to recognize the name Java and favor their browser.

  2. What are the two ways you can run JavaScript?

    1. In the Document Object Model (DOM). In other words, the code runs in the context of the webpage.
    2. Through Node.js, which allows developers to run their JavaScript code in the backend of their webpage.

  3. What was the most popular front-end js library?

    JQuery used to be the most popular front-end JS library, but was limited by it's slow performance.

  4. What is the most popular front-end js library today?

    Now the most popular front-end JS library is React, which was developed by Facebook/Meta.

Engagement Questions March-17

  1. The form tag has an "action" attribute. What does it do?

    The action attribute is where the data is sent (such as a processing.html page) when the user submits the form.

  2. The form tag has a "method" attribute. What does it do?

    The method attribute determines how the data is handled once the user submits with.

  3. What is the difference between a "POST" and a "GET"?

    We use a GET request when solely intend to read, or retrieve data from a server.

    We use a POST request when we are sending data to be stored on the server.
